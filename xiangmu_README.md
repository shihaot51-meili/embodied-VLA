## 💼 项目经历

### pi05 机器人操作策略微调全流程
**时间**：2026.06 - 2026.07 | **角色**：算法工程师

**项目描述**：基于 pi05 基础模型，使用 200~400 条机器人示教数据，完成两个任务（水管插入、主板插线）的微调全流程，实现从数据到部署的端到端管线。

**技术栈**：pi05 VLA 模型 / JAX / FSDP 8卡 / LeRobot 数据格式 / RBY1 机器人

**核心工作**：

**阶段1 — 数据前处理**
- 对原始示教数据进行异常值滤除（clip 到 [-3,3]，清除约 2% 异常帧）
- 实现动作对齐（action = 下一帧 state），消除原始数据中 action 与 state 的不对齐问题
- 主板任务实现控制空间转换（Joint → EE），将 22 维关节向量转为 16 维末端位姿表示

**阶段2 — 数据集合并**
- 实现多批 LeRobot 数据集合并（v4 + v5），自动核对总条数 = 各子集之和
- 设计命名规范：`lerobot_日期_任务_版本_处理标记_机器人_条数s`

**阶段3 — 训练配置编写**
- 设计两个任务的 TrainConfig，命名规范：`pi05_机器人_任务_版本_条数s_from_权重来源_特性_日期`
- 水管任务：Joint 模式 + RTC（双峰延迟采样，max_delay=8），右手操作（arm_joint_mask 逐维度掩码）
- 主板任务：EE 模式（action_space=ee, ee_pose_repr=euler, use_delta_actions 混合策略），左手操作
- 配置训练超参：from base（10万步/1万存）vs from ckpt（2万步/1千存），CosineDecay 学习率调度

**阶段4 — 归一化统计量计算**
- 实现 norm_stats.json 生成（state/action 的 mean/std/min/max/median）
- 解决最易出错的分支：Joint 模式必须 `--torso 6` 跳过躯干，EE 模式禁止加（数据已无躯干维度）
- 保障训练与推理使用同一份 norm 的核心原则

**阶段5 — 模型训练**
- 8 卡 FSDP 并行训练，batch_size=32，XLA 显存控制（90%）
- 监控 loss 曲线：从 ~2.0 平滑下降到 ~0.15，无尖峰毛刺
- 实现 `--overwrite`（全新训练）与 `--resume`（断点续训）两种模式

**项目成果**：
- 两个任务均成功收敛，loss 从 ~2.0 降至 ~0.15
- 水管任务（Joint+RTC）与主板任务（EE）均产出可用 checkpoint
- 建立了可复用的五阶段流水线，支持新任务快速接入

## 📚 技术理解深度

| 领域 | 具体理解 |
|------|---------|
| **正逆运动学** | Joint 模式直接输出关节角度（无需 IK），EE 模式输出末端位姿由控制器做 IK 解算 |
| **动作表示** | 绝对值 vs 增量（delta），混合策略（xyz/euler 增量 + 夹爪绝对值） |
| **姿态表示** | Euler 角（3维，有万向锁风险）vs 四元数（4维，无奇异性） |
| **学习率调度** | Warmup（防初期崩溃）+ CosineDecay（前期快学后期精调），微调用小 lr（2e-5）避免灾难性遗忘 |
| **归一化** | 训练-推理必须用同一份 norm，Joint 模式跳过躯干 6 维，EE 模式不跳过 |
| **微调策略** | 动作专家（Flow Matching 头）为核心微调目标，视觉/语言编码器可冻结保留通用能力 |
| **分布式训练** | FSDP 全分片数据并行，8 卡分摊大模型显存，batch_size=32 |
